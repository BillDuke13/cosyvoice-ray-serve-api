CosyVoice Ray Serve API

An HTTP API that serves the CosyVoice3 text-to-speech model (FunAudioLLM/Fun-CosyVoice3-0.5B-2512) through Ray Serve, with autoscaling replicas, automatic GPU detection, reference-audio caching, MP3 output, and optional response streaming.

The whole service — the CosyVoiceService deployment, the AudioProcessor helpers, the Starlette routing, and the cosyvoice_app entry point — lives in a single module, api.py. The cosyvoice/ package is vendored from upstream CosyVoice commit 074ca6dc9e80a2f424f1f74b48bdd7d3fea531cc and is treated as read-only.

Capabilities

  • Standard TTS with a built-in voice — /tts clones a bundled reference clip selected by voice_type.
  • Zero-shot voice cloning — clone a voice from an uploaded reference clip plus its transcript.
  • Cross-lingual synthesis — clone from an uploaded clip with no transcript; the target text may be in another language.
  • Instruction-controlled synthesis — a natural-language instruction steers style, dialect, speed, or emotion over a built-in voice.
  • CosyVoice3 prompt compatibility — legacy client text is wrapped internally with <|endofprompt|> where the model requires it.
  • Streaming — pass "stream": true for a chunked audio/mpeg response.
  • Adjustable speed — optional finite speed factor from 0.5 to 2.0 on every synthesis endpoint.

Quick start

Install dependencies (Python 3.10; pynini from conda-forge first), then run the service against a Ray cluster:

conda create -n cosyvoice-api python=3.10 -y
conda activate cosyvoice-api
conda install -y -c conda-forge pynini==2.1.5
pip install -r requirements.txt

serve run api:cosyvoice_app        # or: python api.py (local single-node cluster)

The model FunAudioLLM/Fun-CosyVoice3-0.5B-2512 auto-downloads from ModelScope into pretrained_models/Fun-CosyVoice3-0.5B/ on first startup. The HTTP proxy binds to SERVE_PORT (default 8000 locally). Then synthesize:

curl -X POST http://localhost:8000/tts \
  -H 'Content-Type: application/json' \
  -d '{"text": "Hello, world.", "voice_type": "qwen"}' \
  --output hello.mp3
Audio sample rates are fixed and load-bearing: model output is 24 kHz and reference/prompt audio is normalized to 16 kHz mono.
