API Reference

HTTP endpoints served by the Starlette app in api.py. The base URL is http://<host>:<SERVE_PORT>; SERVE_PORT defaults to 8000 for local runs and is 9998 in the Docker image.

All four synthesis endpoints return an MP3 (Content-Type: audio/mpeg). Add "stream": true for a chunked audio/mpeg response, and an optional speed (finite float, 0.5 to 2.0; 1.0 = normal). The corresponding CosyVoiceService method in api.py is the source of truth for request validation.

Synthesis endpoints

POST /tts - standard TTS

Synthesizes text with a built-in voice by cloning the bundled reference clip named by voice_type via inference_cross_lingual.

Body: application/json

FieldTypeRequiredDefaultDescription
textstringyes-Text to synthesize
voice_typestringnoqwenBuilt-in voice key from VOICE_PROMPTS; unknown values fall back to the default
speednumberno1.0Speech-rate factor (0.5 to 2.0)
streambooleannofalseStream the response in chunks
curl -X POST http://localhost:8000/tts \
  -H 'Content-Type: application/json' \
  -d '{"text": "Hello, world.", "voice_type": "qwen", "speed": 1.0}' \
  --output out.mp3

POST /zero_shot_tts - zero-shot cloning

Clones a voice from an uploaded reference clip and its transcript (inference_zero_shot). Plain reference_text is wrapped in the CosyVoice3 prompt format if it does not already include <|endofprompt|>.

Body: multipart/form-data

FieldTypeRequiredDescription
textstringyesText to synthesize
reference_textstringyesTranscript of the reference clip
reference_audiofileyesReference audio (WAV/MP3/...); normalized to 16 kHz mono
speednumbernoSpeech-rate factor (0.5 to 2.0; default 1.0)
streambooleannotrue for streaming
curl -X POST http://localhost:8000/zero_shot_tts \
  -F 'text=This voice is cloned.' \
  -F 'reference_text=Transcript of the sample.' \
  -F 'reference_audio=@sample.wav' \
  --output out.mp3

POST /cross_lingual_tts - cross-lingual cloning

Clones a voice from an uploaded clip with no transcript; the target text may be in a different language (inference_cross_lingual). Plain text is wrapped internally with the required CosyVoice3 prompt marker.

Body: multipart/form-data

FieldTypeRequiredDescription
textstringyesText to synthesize (any supported language)
reference_audiofileyesReference audio; normalized to 16 kHz mono
speednumbernoSpeech-rate factor (0.5 to 2.0; default 1.0)
streambooleannotrue for streaming
curl -X POST http://localhost:8000/cross_lingual_tts \
  -F 'text=你好，世界。' \
  -F 'reference_audio=@sample.wav' \
  --output out.mp3

POST /instruct_tts - instruction-controlled TTS

Steers style, dialect, speed, or emotion with a natural-language instruction over a built-in voice (inference_instruct2; the voice_type clip supplies the timbre). If instruction does not include <|endofprompt|>, the service wraps it as You are a helpful assistant. {instruction}<|endofprompt|>.

Body: application/json

FieldTypeRequiredDefaultDescription
textstringyes-Text to synthesize
instructionstringyes-Natural-language instruction
voice_typestringnoqwenBuilt-in voice supplying the timbre
speednumberno1.0Speech-rate factor (0.5 to 2.0)
streambooleannofalseStream the response in chunks
curl -X POST http://localhost:8000/instruct_tts \
  -H 'Content-Type: application/json' \
  -d '{"text": "Today is a wonderful day.", "instruction": "Speak cheerfully."}' \
  --output out.mp3

Operational endpoints

GET /health

Liveness/readiness check. Also reachable at /v1/model/cosyvoice/healthcheck (the Docker HEALTHCHECK target). Returns 200 when the model is loaded, otherwise 503.

{
  "status": "healthy",
  "timestamp": "2026-05-31T12:00:00+0000",
  "device": "cuda:0",
  "model_loaded": true,
  "ray_actor_id": "...",
  "ray_node_id": "..."
}

GET /config

Returns the active service configuration.

{
  "default_voice": "qwen",
  "available_voice_prompts": ["qwen"],
  "sample_rate": 24000,
  "reference_audio_sample_rate": 16000,
  "device_in_use": "cuda:0",
  "model_directory": "/app/pretrained_models/Fun-CosyVoice3-0.5B",
  "model_id": "FunAudioLLM/Fun-CosyVoice3-0.5B-2512",
  "model_dir_name": "Fun-CosyVoice3-0.5B",
  "model_revision": "main",
  "model_family": "cosyvoice3",
  "max_replicas_config": "2"
}

Errors

Errors are returned as JSON. Handled HTTPExceptions use their own status code and {"detail": "<message>"}; unhandled server errors return 500 with {"detail": "Internal server error"}.

StatusWhen
400Invalid JSON, missing/invalid fields, a non-finite or out-of-range speed, or an unavailable voice prompt
500Synthesis or audio-encoding failure
503/health when the model is not loaded
CORS is enabled for all origins, methods, and headers (see the CORSMiddleware configuration in api.py); restrict it before exposing the service publicly.
