Architecture

How a request becomes audio, how CosyVoice3 is driven, and how the Ray Serve deployment is configured. Everything below lives in api.py; the cosyvoice/ package is the vendored upstream model source.

Module layout

  • AudioProcessor - stateless helpers: normalize a reference clip to 16 kHz mono WAV (FFmpeg), normalize/validate audio tensors, and encode a tensor to MP3 (soundfile WAV to FFmpeg LAME).
  • CosyVoiceService - the @serve.deployment class: downloads and loads the model per replica, maps each endpoint to an AutoModel inference method, and handles streaming/non-streaming responses.
  • Starlette app (cosyvoice_app) - routes, exception handlers, and CORS. This is the object serve run api:cosyvoice_app resolves.

Request lifecycle

  1. An HTTP request hits a Starlette Route, which calls the bound deployment method (for example, tts_standard).
  2. The handler parses and validates input, resolves voice_type, speed, and stream, prepares prompt audio, and adds the CosyVoice3 <|endofprompt|> marker when legacy clients omit it.
  3. It calls _generate_audio_common(method_name, text, stream, **kwargs), which looks up the model method and drives it with the same stream mode requested by the client. Non-streaming requests keep model-level speed control active.
  4. Each returned tensor is validated and peak-normalized. For a streaming response, every chunk is encoded to MP3 and yielded as it is produced; otherwise chunks are concatenated into one tensor and encoded once.
  5. The handler returns a StreamingResponse or a buffered Response with media_type="audio/mpeg". Temporary files in tmp/ are cleaned up afterward.

AutoModel detects cosyvoice3.yaml in the local model directory and constructs CosyVoice3. vLLM and TensorRT flags are left off by default so this upgrade changes the model family without changing inference runtimes.

CosyVoice3 prompt handling

  • /tts and /cross_lingual_tts wrap plain text as You are a helpful assistant.<|endofprompt|>{text}.
  • /zero_shot_tts wraps plain reference_text as You are a helpful assistant.<|endofprompt|>{reference_text}.
  • /instruct_tts wraps plain instruction as You are a helpful assistant. {instruction}<|endofprompt|>.
  • Advanced clients may send prompts that already include <|endofprompt|>; the service leaves those strings unchanged.

Audio pipeline

Two sample rates are fixed and load-bearing:

ConstantValueRole
SAMPLE_RATE24000CosyVoice3 synthesis output rate
REFERENCE_AUDIO_SAMPLE_RATE16000Prompt/reference audio rate (mono)
  • Reference normalization - uploaded and built-in clips are converted to 16 kHz mono WAV via FFmpeg (process_reference_audio). The normalized path is passed to upstream CosyVoice, which loads the prompt audio as needed.
  • MP3 encoding - output tensors are written to a temporary WAV with soundfile (PCM_16), then encoded to MP3 with FFmpeg's LAME encoder at VBR quality 2.
  • Caching - processed reference clips are cached by path + mtime (_reference_audio_cache); built-in voice prompt paths are cached by voice_type.
  • Temporary files - intermediate WAV/MP3 files live in tmp/ and are removed in finally blocks after each request.

Ray Serve deployment

The @serve.deployment decorator (deployment name CosyVoiceAPI) configures behavior from environment variables; see the deployment guide for the full list:

  • Autoscaling - min/initial/max replicas and a target of concurrent requests per replica.
  • Resources - ray_actor_options={"num_gpus": 1} when CUDA is available and NUM_GPUS_PER_REPLICA > 0; otherwise no GPU is requested.
  • Health and load shedding - health-check period/timeout, plus max ongoing and max queued requests.
  • Graceful shutdown - timeout and poll interval for draining in-flight requests.
  • Dynamic config - user_config seeds default voice, sample rate, and model metadata; reconfigure() applies live updates such as a new default voice.

Per-replica startup

  1. _setup_device() reads ray.get_gpu_ids() to pick cuda:<id>, falling back to cpu when no GPU is assigned.
  2. _download_and_setup_models() downloads FunAudioLLM/Fun-CosyVoice3-0.5B-2512 from ModelScope into pretrained_models/Fun-CosyVoice3-0.5B/ if required files are absent, then constructs AutoModel(load_trt=False, load_vllm=False, fp16=<on GPU>).
  3. If GPU initialization fails, the replica retries on CPU (without FP16) before giving up.
  4. An atexit handler releases the model, clears cached reference audio, and clears the CUDA cache on shutdown.
